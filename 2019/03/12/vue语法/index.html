自定义指令

自定义指令
0 => 试验得出 自定义指令的名称不可以用大写 官网不表明 你奶奶
1 => 自定义指令是一个独立的成员,并不在vm中创立,在vm实例外创建的指令叫全局指令
2 => 语法/固定写法 => Vue.directive(“指令名称”,{配置})
3 => 自定义指令的勾子函数
3.1 bind : 只调用一次,指令第一次绑定到元素时调用。
3.2 inserted ：被绑定元素插入父节点时调用/就是给dom元素添加这个指令生成时触发
3.3 update ：所在组件的VNode更新时调用
3.4 componentUpdated ：指令所在组件的VNode及其子VNode全部更新后调用
3.5 unbind : 只调用一次,指令与元素解绑时调用
4 => 指令勾子函数会默认传入几个参数
4.1 el : 指令所绑定的元素,可以直接操作dom/谁添加了这个指令谁就是这个el
4.2 bindind : 是一个对象
4.2.1 包含的属性 => value指令的绑定值
5 => 示范
Vue.directive(“getFocus”,{
inserted(el){
el.focus();
}
})
6 => 调用自定义指令,直接在需要调用的dom元素上添加 v-指令名称 就行
7 => 好处就是 ref的写法就是固定写死 自定义指令 谁要谁调

实现-使用自定义指令来自动获取光标焦点 - 改变颜色

自定义过滤器

自定义过滤器 => 功能就是对数据进行过滤操作,本质上是一个函数
1 在vm外创建的过滤器称为全局过滤器,在组件内部创建的过滤器称为局部过滤器
2 语法 => Vue.filter(“过滤器名称”, (value)=>{})
2.1 通过在dom标签中使用管道符来调用过滤器 => 源数据 | 过滤器名称
2.2 过滤器默认会将源数据作为参数传递也就是value - 就是把dom中的管道符前面的数据作为数据传递
2.3 示范 =>
Vue.filter(“setDate”,function(value){
console.log(value)
})

axios Github官网
axios需要引入一个axios的js文件 详细情况在github官网有说明
axios中的get请求参数 必须存入params对象中 也就是 => { params: { id:1, name:hello }}
axios中的post请求参数 存入 { id:1 , name:jack } 这样一个对象就可以了 不用想get.axios 请求一样需要传参进params中

vue-resource

vue中的axios不能实现跨域请求 只能使用vue中的resource来实现跨域
Github地址vue-resource
1. 通过 https://cdn.jsdelivr.net/vue.resource/1.2.1/vue-resource.min.js 下载到vue-resource文件
2. 在html页面中通过script标签导入vue-resource.min.js 文件后，就会自动的在Vue对象实例上初始化 $http
2.1 说明 => 把第1的链接地址打开 然后ctrl+A 全选把代码复制到一个js文件中保存起来 然后在需要用到jsonp实现跨域请求的html页面中引入 这个js文件
2.2 说明 => 当我们手动把resource文件引入时 就会在vue中的vm实例上自动绑定 $http
3. 全局Vue构造器写法
3.1 => 全局Vue构造器写法 , 在勾子函数里面的写法 mounted(){}
Vue.http.jsonp(“url,{参数}”)
.then (
3.2 .then里面的回调 有两个回调函数 一个成功的回调 一个失败的回调
(result) => { console.log(result) }
(error) => { console.log(error) }
)
4. 在Vue实例中的写法 , 也就是在methods自定义函数区域里面的写法
this.$http.jsonp(“url”,{参数})
.then (
(result) => { console.log(result) }
(error) => { console.log(error) }
)
