自定义指令

自定义指令
0 => 试验得出 自定义指令的名称不可以用大写 官网不表明 你奶奶
1 => 自定义指令是一个独立的成员,并不在vm中创立,在vm实例外创建的指令叫全局指令
2 => 语法/固定写法 => Vue.directive(“指令名称”,{配置})
3 => 自定义指令的勾子函数
3.1 bind : 只调用一次,指令第一次绑定到元素时调用。
3.2 inserted ：被绑定元素插入父节点时调用/就是给dom元素添加这个指令生成时触发
3.3 update ：所在组件的VNode更新时调用
3.4 componentUpdated ：指令所在组件的VNode及其子VNode全部更新后调用
3.5 unbind : 只调用一次,指令与元素解绑时调用
4 => 指令勾子函数会默认传入几个参数
4.1 el : 指令所绑定的元素,可以直接操作dom/谁添加了这个指令谁就是这个el
4.2 bindind : 是一个对象
4.2.1 包含的属性 => value指令的绑定值
5 => 示范
Vue.directive(“getFocus”,{
inserted(el){
el.focus();
}
})
6 => 调用自定义指令,直接在需要调用的dom元素上添加 v-指令名称 就行
7 => 好处就是 ref的写法就是固定写死 自定义指令 谁要谁调

实现-使用自定义指令来自动获取光标焦点 - 改变颜色

自定义过滤器

自定义过滤器 => 功能就是对数据进行过滤操作,本质上是一个函数
1 在vm外创建的过滤器称为全局过滤器,在组件内部创建的过滤器称为局部过滤器
2 语法 => Vue.filter(“过滤器名称”, (value)=>{})
2.1 通过在dom标签中使用管道符来调用过滤器 => 源数据 | 过滤器名称
2.2 过滤器默认会将源数据作为参数传递也就是value - 就是把dom中的管道符前面的数据作为数据传递
2.3 示范 =>
Vue.filter(“setDate”,function(value){
console.log(value)
})
