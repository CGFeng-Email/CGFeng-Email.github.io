什么时候能用rpx呢 就是 设计稿的宽度为 750px时 才可以使用

程序的生命周期函数

1: 在程序入口app.js中调用App(), 而且必须调用且只能调用一次，然后再App()的参数中可以声明生命周期函数
2: 后台：当用户点击右上角关闭，或者按了设备 Home 键离开微信，此时小程序并没有直接销毁，而是进入了后台
3: 函数 => onLaunch , 描述 => 监听小程序初始化 , 触发时机 => 小程序初始化完成时（全局只触发一次）, 从当前小程序离开5分钟后就会销毁 , 所以生命周期为5分钟
4: 函数 => onShow , 描述 => 监听小程序显示/代码解释完毕 渲染页面 , 触发时机 => 小程序启动，或从后台进入前台显示时,代码解析完毕,渲染页面执行
5: 函数 => onHide , 描述 => 监听小程序隐藏/也就是从小程序切换到了QQ聊天这么一个过程 , 触发时机 => 小程序从前台进入后台时
示例 程序的生命周期是写在全局的js里面 写在这个app({}) 对象里面 不能删掉

vue区块-详解

发表于 2019-03-02 | 分类于 vue区块-详解

el => 映射范围

是vue实例的映射范围

data => 普通数据值

是一个对象
用来写属性值

methods => 自定义函数

是一个对象
专门用来声明自定义函数

mounted => 勾子函数

是一个函数 => mounted(){}
勾子函数/自动触发的函数/页面一打开就立刻触发的函数/相当于window.onload

computed => 计算属性

计算属性 => 基于缓存 => 能用computed 就不要用 methods
是一个属性 => computed: {} 在里面定义一些函数 一定得是依赖的数据不发生改变时是不会触发的 也就是 this. 发生变化时才会触发响应
不会传递参数
通过this.XXX 拿到data中的属性值
优点: 可以在dom元素上通过 v-model=”来调用computed计算属性区中定义的函数”
优点2: conputed中定义的函数 带有缓存 只有在所依赖的数据发生改变时才会触发调用这个函数
缺点: 异步操作时不会调用computed里面自定义的函数 如: 在computed里面写一个定时器 再在定时器里面定义函数时 是不会触发这个函数的

watch => 监听

watch 是一个对象
watch中成员的名称不能随意定义,它的名称只能和data中的成员名称一致。你想监听data中哪个成员的值的变化,就添加这个成员名称的函数
它的函数有两个参数 new：新的值 , old: 旧的值
默认情况下 watch监听只能监听一层 如 => obj: { name : “jack” } 当想监听obj.name 时就无法做到监听 也就是只能监听obj这个对象
要想实现深度监听 必须添加 handler(newV,oldV){} 函数
watch里面有一个默认的属性值 deep: false; 当把这个值改为true时才会实现深度监听
watch的深度监听是监听的对象里面任何数据发生改变时都会执行响应的 这样并不好 要想实现深度监听指定的对象值 可以用 “” 包起来作为函数的名字进行监听

vue指令-详解

v-text=”值” 只会把data里面的值原输出
v-html=”值 当data里面的值是 标签格式则会解析标签 即为 生成标签样式
v-bind: 它的功能是让冒号后面的值 进行动态的绑定 也是属于单向绑定 v-bind:绑定如果model中的数据发生改变了 view中的数据也会跟着发生改变 但是反过来view中的数据发生改变 model中的数据却未必会跟着改变
v-for=”(value,index) in 目标”
v-model=”” 双向绑定 即是 model中的数据发生改变 view中的展示数据也会发生改变 反过来也是如此
v-show=”值” 控制标签元素的显示与隐藏 生成标签 相当于添加了 display：none
v-if=”值” 控制的是属性标签 显示与隐藏 如果值为false 不生成标签接口
v-on:click=”事件处理函数(参数)” 缩写 => @click=”” 动态绑定一个处理函数 自定义函数 写在methods区块
v-once 只会执行一次的指令

jquery的属性方法

控制(修改)css的样式  -  以传入一个{}大括号对象的形式作为参数 里面的属性值为 键值对 就可修改多个属性值
    $("div").css( { width:200, height: 200, backgroundColor: "pink" } )

获取目标元素的某一个值 并且弹出 - 如果只传入一个属性值 就是获取该属性的值
     var width =  $("div").css( "width" );
     alert( width );

 添加css类 - 添加的css类 不需要添加. 或者# 并且可以添加多个类 只要空格隔开就行
    $("div").addClass("box");

 移除css类
    $("div").removeClass("box");

检测是否添加了css类 - 有则返回true 没有则返回false
    $("div").hasClass("box");

切换css类 - 原理：添加类 / 移除类 - 可同时添加多个类或移除多个类
    $("div").toggleClass("box");

.eq(索引值) 索引 - 获取到索引为几的元素
    $("div").eq(0).addClass("box");  当有两个div时 只给第一个div添加box样式

$(this)代表当前

.siblings()选中兄弟元素 使用siblings实现jq排他效果

    <body>
        (button{$})*5   添加5个button元素
    </body>

    jq入口函数
    $(function () {
        注册事件
        $("button").click(function (){
        $(this) 代表当前
        $(this).css( {backgroundColor: "pink"} );
        .siblings() 只会选中兄弟元素
        $(this).siblings().css( {backgroundColor: ""} )
    })
  })

.parent() 父级关系 通过函数形式查找父级
    $(this).parent().parent() - 找到当前元素的父级的父级

.parents( "body/html" ) 祖先 - 通过函数的形式查找到祖先 - 需要写入查找的祖先值

.children() 子级 - 通过函数的形式获取子级元素
    $("div").children(".son")  当有两个子元素时 可直接输入需要获取的子元素

.index() 索引 - 通过函数的形式获取到索引值
    $(this).index(); - 获取当前元素的索引值

GITHUB

发表于 2019-02-24 | 分类于 Github项目上传详情

1 先配置用户名和邮件

git config --global user.name xxx   配置 用户名    
git config --global user.email xxx   配置 邮箱
git config user.name 查看用户名
git config user.email 查看邮箱

2 在项目的根目录下 打开git bash shell

1 git init  初始化 git的仓库 成功之后 会在根目录下多了一个隐藏的文件夹 .git 该文件夹便是存放版本数据的地方 千万不要手动去编辑这个文件
2 查看git仓库的状态  git status
3 添加所有文件 git add .
4 再次查看git status的状态 看是否有未添加的文件
5 把git仓库里面的文件提交到本地仓库 git commit -m "操作说明"
6 查看仓库的状态 git status
7 git remote add origin url  origin是一个变量名可随便修改 把远程仓库的地址 存放到一个变量里面 url远程仓库的地址
8 git push -u origin master  把本地仓库推送到远程仓库  第二次推送的时候 只需要 git push 即可

3 从github克隆下来的项目 获取最新代码

git pull 拉取更新
